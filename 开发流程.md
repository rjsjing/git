\# 待办事项应用开发全流程



\## 第一步：项目初始化

\- 在本地桌面创建 `git` 文件夹

\- 初始化 Git 仓库（`git init`）

\- 在 GitHub 创建远程仓库并关联本地



\## 第二步：配置 AI 编程环境

\- 安装 Claude Code

\- 接入 DeepSeek API

\- 配置 Git 用户信息



\## 第三步：搭建 Node.js 项目骨架

\- 创建 `package.json`，声明启动命令和依赖

\- 创建 `.gitignore`，排除 `node\\\_modules/`

\- 执行 `npm install` 安装 Express



\## 第四步：编写后端 REST API

\- 创建 `server.js`，定义 5 个 API 端点：

&#x20; - GET /api/todos（查询，支持 ?q= 搜索）

&#x20; - GET /api/todos/:id（获取单个）

&#x20; - POST /api/todos（新增）

&#x20; - PUT /api/todos/:id（更新）

&#x20; - DELETE /api/todos/:id（删除）

\- 数据暂存内存数组



\## 第五步：编写前端界面

\- 创建 `public/todo.html`，包含 HTML+CSS+JS

\- 实现添加、勾选完成/未完成、删除、搜索功能

\- 所有数据操作通过 `fetch` 调用后端 API



\## 第六步：前后端对接

\- 在 `server.js` 中添加 `app.use(express.static('public'))`

\- 修改前端，将 `localStorage` 替换为 API 调用

\- 启动后访问 `http://localhost:3000/todo.html` 使用



\## 第七步：版本控制与发布

\- 使用 Git 提交所有代码

\- 推送至 GitHub 仓库 `https://github.com/rjsjing/git`



\## 涉及的核心概念

| 概念 | 作用 |

|------|------|

| Git | 版本控制，记录修改历史 |

| GitHub | 代码托管和协作平台 |

| Node.js | 服务器端 JavaScript 运行环境 |

| npm | 包管理器，安装 Express 等依赖 |

| Express | Web 框架，简化后端路由和请求处理 |

| REST API | 使用 HTTP 方法的增删改查规范 |

| JSON | 前后端数据交换格式 |

| 静态文件托管 | Express 直接提供 HTML/JS/CSS |

| fetch + async/await | 前端调用后端接口的现代方式 |



\## 架构图

用户浏览器 → Express 服务器（静态文件 + REST API） → 数据存储（内存/数据库）
